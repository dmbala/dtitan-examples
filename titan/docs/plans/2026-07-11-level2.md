# TorchTitan Workshop — Level 2 Implementation Plan

**Goal:** Write the participant-facing **Level 2** module — composable
parallelism (2D FSDP2+TP, context parallel), memory diagnostics (activation
checkpointing, memory snapshots), the flagship DCP save/resume restart lab,
and a real-8B taste — plus its GPU-validation checklist. This is an
**operate-the-framework** deliverable: config + launcher commands + guided
docs, not new Python model code or a TDD task list. All commands and expected
outputs below were already run and confirmed on GPU hardware; this plan is the
authoring/organizing pass over that validated output, not a build-and-test
loop.

**Architecture:** Level 2 reuses the Level 1 scaffolding unchanged — same
container (`dtitan-torch211.sif`), same launcher (`slurm/launch_1node.sbatch`,
1 node / 4 GPUs on `kempner_rtx`), same debug tokenizer and C4 subset. What's
new is entirely on the **config side**: additional dotted CLI overrides
(`--parallelism.tensor_parallel_degree`, `--parallelism.context_parallel_degree`,
`--activation_checkpoint.mode`, `--profiling.enable_memory_snapshot`,
`--checkpoint.*`) composed on top of the same `--model.name llama3
--model.flavor debugmodel` entrypoint, plus one run with `--model.flavor 8B`
and the real Llama-3.1 tokenizer. No new library code, no custom config
registry entries — every flag used below is a built-in TorchTitan config path
exercised through the existing launcher.

**Tech Stack:** TorchTitan 0.2.2, PyTorch 2.11.0a0 (CUDA 13.2), torchao,
Slurm, Apptainer/Singularity — identical to Level 1's Global Constraints.

## Global Constraints

- **Container:** `dtitan-torch211.sif`
  (`/n/holylfs06/LABS/kempner_shared/Everyone/containers/applications/dtitan/dtitan-torch211.sif`),
  torch 2.11.0a0 / CUDA 13.2 / torchtitan 0.2.2 — same image as Level 1.
- **Launcher:** `slurm/launch_1node.sbatch` (1 node / **4 GPUs**, `kempner_rtx`,
  account `kempner_dev`). Level 2 does not need `launch_8gpu.sbatch` or
  `launch_2node.sbatch` — every mesh here is a 2×2 composition on 4 GPUs.
- **Assets:** debug tokenizer `--model.hf_assets_path=assets/test_tokenizer`
  (vocab 2016, matches `debugmodel`'s `vocab_size=2048`); dataset
  `--training.dataset=c4_test --training.dataset_path=assets/c4_subset`,
  generated once via `python scripts/prepare_c4_subset.py`. Real tokenizer +
  trainable target for the 8B lab: `$MODELS=/n/holylfs06/LABS/kempner_shared/Everyone/testbed/models`,
  `--model.hf_assets_path=$MODELS/Llama-3.1-8B-Instruct` — valid only with
  `--model.flavor 8B`.
- **Verified override paths (Level 2, in addition to Level 1's):**
  `--parallelism.tensor_parallel_degree`, `--parallelism.context_parallel_degree`,
  `--activation_checkpoint.mode` (`selective`/`full`/`memory_budget`/`none`),
  `--profiling.enable_memory_snapshot`, `--checkpoint.enable`,
  `--checkpoint.interval`, `--checkpoint.no-last-save-model-only`,
  `--model.flavor 8B`. Boolean flags use the `--no-<flag>` negation form —
  `--checkpoint.last_save_model_only=false` is rejected
  (`Unrecognized options: false`).
- **The parallel-dims rule (from Level 1) still applies:** `dp_replicate *
  dp_shard * cp * tp * pp == WORLD_SIZE` (4 here). Every Level 2 mesh below
  multiplies out to 4.
- **The DCP restart gotcha:** `checkpoint.last_save_model_only` defaults to
  `True`, so the *last* checkpoint TorchTitan writes is model-only and not
  resumable — resuming from it fails with `RuntimeError: Missing key in
  checkpoint state_dict: dataloader.dp_rank_0`. Every checkpointed run in this
  level passes `--checkpoint.no-last-save-model-only` to avoid it, and the
  restart lab (L2.5) teaches the failure deliberately.
- **Blackwell caveat (from Level 1):** logs warn `Peak flops undefined for RTX
  PRO 6000 Blackwell, fallback to A100` — MFU is computed against the A100
  peak-FLOPs table, so treat MFU% as indicative only.
- **Working directory** for all commands is `titan/`.

|-----|---------------------------|-------------------|

| L2.5 — DCP save + resume | SAVE: `--checkpoint.enable --checkpoint.interval=5 --checkpoint.no-last-save-model-only --training.steps=10`; RESUME: same flags, `--training.steps=15` | `step-5`/`step-10` written; resume logs `Loading the checkpoint from …/checkpoint/step-10.` → `Finished loading … in 0.24 seconds.` → completes at step 15; teaches the `Missing key ... dataloader` gotcha |
