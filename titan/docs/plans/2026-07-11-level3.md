# TorchTitan Workshop — Level 3 Implementation Plan

**Goal:** Author a complete, runnable **Level 3** module — production-scale parallelism composition (HSDP+TP 3D mesh, Pipeline Parallel, Mixture-of-Experts + Expert Parallel), FP8 quantization, and NCCL/Flight-Recorder debugging — driven entirely through TorchTitan's existing `--model.name`/`--model.flavor`/`--parallelism.*`/`--comm.*`/`--model.converters` CLI-override interface, on the already-built `slurm/launch_8gpu.sbatch` launcher.

**Architecture:** Same as Level 1: this workshop *operates* TorchTitan rather than adding model code. Level 3 deliverables are **docs + config overrides**, not Python. All commands compose flags that are already GPU-validated (individually, and in several cases in exactly the combinations used here) on `kempner_rtx` with the `dtitan-torch211.sif` container — no new launcher, no new preflight script, no new tests. The only new artifacts are three documentation files.

**Tech Stack:** TorchTitan 0.2.2, PyTorch 2.11.0a0 (rebuilt container), torchao 0.17, Slurm, Apptainer/Singularity — unchanged from Levels 1–2.

## Global Constraints

- **Container:** `dtitan-torch211.sif` — `IMAGE=/n/holylfs06/LABS/kempner_shared/Everyone/containers/applications/dtitan/dtitan-torch211.sif`. Same image as Levels 1–2; no rebuild needed for Level 3.
- **Launcher:** `slurm/launch_8gpu.sbatch` — 1 node, **8 GPUs**, entrypoint `--model.name llama3 --model.flavor debugmodel "$@"` (last-occurrence override wins, so `--model.name deepseek_v3 --model.flavor debugmodel` in `"$@"` replaces it for the MoE lab). Already sets the writable HF cache, `TRITON_LIBCUDA_PATH=/.singularity.d/libs`, and `TORCH_FR_BUFFER_SIZE`. `slurm/launch_2node.sbatch` is out of scope for the required labs — it exists only for the >8-GPU planning demos.
- **Config interface (verified in-container, Level 3 surface):** `--parallelism.data_parallel_replicate_degree`, `--parallelism.data_parallel_shard_degree`, `--parallelism.tensor_parallel_degree`, `--parallelism.pipeline_parallel_degree`, `--parallelism.expert_parallel_degree`, `--model.converters quantize.linear.float8`, `--comm.trace_buf_size`. The parallel-dims assertion is `dp_replicate*dp_shard*cp*tp*pp == WORLD_SIZE`; `expert_parallel_degree` is **not** in that product (orthogonal, applied within dp ranks) — on 8 GPUs the valid MoE combination is `dp_shard=8 + ep=2`, not `dp_shard=4 × ep=2`.
- **Assets (offline, unchanged from Levels 1–2):** debug tokenizer `--model.hf_assets_path=assets/test_tokenizer` (vocab 2016, matches `llama3 debugmodel`'s `vocab_size=2048` **and** `deepseek_v3 debugmodel`'s vocab 2048 — no separate MoE tokenizer needed). Training data: one-time `python scripts/prepare_c4_subset.py`, then `--training.dataset=c4_test --training.dataset_path=assets/c4_subset`.
- **Cluster:** `--account=kempner_dev`, `--partition=kempner_rtx` (RTX PRO 6000 Blackwell, 8 GPUs/node), QoS `kempner_base`, 16-GPU/user cap. Every Level 3 lab uses exactly 8 of those GPUs, on one node.
- **Working directory** for all commands is `titan/`.
- **Scope discipline:** no new Python model/library code (this is the "operate the framework" level); no edits to existing launchers, tests, spec, or Level 1/2 docs; three new doc files only.

## Deliverables

### 1. `titan/docs/labs/level3/README.md`

Participant-facing guide: overview (hardware, common overrides, why 8-GPU-single-node), 5 labs, a planning-only scaling-demo note, a capstone, and a troubleshooting table.

| Lab | Teaches | Validated command | Validated result |
|-----|---------|--------------------|-------------------|
| L3.1 HSDP+TP | Real 3D mesh (`dp_replicate=2 × dp_shard=2 × tp=2`) | `sbatch slurm/launch_8gpu.sbatch --model.hf_assets_path=assets/test_tokenizer --training.dataset=c4_test --training.dataset_path=assets/c4_subset --training.seq_len=512 --training.local_batch_size=8 --parallelism.data_parallel_replicate_degree=2 --parallelism.data_parallel_shard_degree=2 --parallelism.tensor_parallel_degree=2 --training.steps=20` | mesh `dp_replicate=2, dp_shard=2, cp=1, tp=2`; loss step 10 ≈ 6.22 → step 20 ≈ 3.57; completed |
| L3.2 Pipeline Parallel | Last-stage-only real loss; sentinel loss on other ranks | `… --parallelism.pipeline_parallel_degree=2 --parallelism.data_parallel_shard_degree=4 --training.steps=20` | completes; non-last-stage sentinel loss ≈ `-4.00000`; last-stage loss ≈ 3.61 |
| L3.3 MoE + EP | `ep` orthogonal to world-size product; `TRITON_LIBCUDA_PATH` requirement | `sbatch slurm/launch_8gpu.sbatch --model.name deepseek_v3 --model.flavor debugmodel --model.hf_assets_path=assets/test_tokenizer --training.dataset=c4_test --training.dataset_path=assets/c4_subset --training.seq_len=512 --training.local_batch_size=8 --parallelism.data_parallel_shard_degree=8 --parallelism.expert_parallel_degree=2 --training.steps=20` | loss step 10 ≈ 4.48 → step 20 ≈ 3.47, memory ≈ 2.2 GiB; completed |
| L3.4 FP8 | Converter name `quantize.linear.float8`; SM89+/Blackwell sm_120 via torchao | `… --model.converters quantize.linear.float8 --parallelism.data_parallel_shard_degree=8 --training.steps=10` | loss step 1 ≈ 8.02 → step 10 ≈ 4.62; completed |
| L3.5 Flight Recorder | `TORCH_FR_BUFFER_SIZE` + `--comm.trace_buf_size`; dump-on-hang framing | `… --comm.trace_buf_size=20000 --training.steps=6` | completes; comm traces under `outputs/<...>/comm_traces/` |

Plus:
- A **scaling/architecture demos (planning only)** note for `deepseek_v3` `16B`/`236B`/`671B` and `llama3` `70B`/`405B` — config-inspection-only, no `sbatch` submission.
- A **capstone**: HSDP+TP (L3.1) + a DCP checkpoint (`--checkpoint.enable --checkpoint.interval --checkpoint.no-last-save-model-only`, the resumable-checkpoint pattern validated in Level 2) + participant's choice of FP8 (L3.4) or MoE (L3.3), with an explicit honesty note that the individual flags are each validated but the specific combined capstone command was not independently re-run end-to-end.
